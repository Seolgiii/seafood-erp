# 운영 환경 검증 체크리스트 — 동결비 + 옵션 B + 이월 경비

배포 후 운영 환경에서 골든패스 1회 돌려 데이터 흐름 확인.

작업 기준일: 2026-05-13
검증 LOT: 신규 입고로 만든 테스트 LOT 권장 (기존 LOT 영향 없음)

---

## 사전 확인

- [ ] Airtable LOT별 재고 테이블에 `_tmp_삭제대상` suffix 필드 4개 모두 삭제됨
- [ ] LOT.이동입고일 필드 존재 (date, ISO)
- [ ] LOT.이월냉장료/이월입출고비/이월노조비/이월동결비 필드 존재 (currency, precision 0)
- [ ] 출고 관리.출고시점 동결비 필드 존재 (currency, precision 2)
- [ ] 보관처 비용 이력 테이블에 모든 운영 보관처의 동결비가 채워져 있음

---

## 시나리오 1 — 입고 신청 → 승인

**액션**: 한림냉동에 고등어 10박스 신규 입고 신청 → 관리자 승인

**확인 포인트 (LOT별 재고)**:
- [ ] 최초입고일 = 입고일 (예: 2026-05-13)
- [ ] **이동입고일 빈칸** ← 핵심 (이전 마이그레이션 롤백 검증)
- [ ] 재고수량 = 10
- [ ] 냉장료단가 / 입출고비 / 노조비 / **동결비** = 보관처 비용 이력 값과 일치

**예상값** (한림냉동 기준):
- 냉장료단가 = 1500
- 입출고비 = 500
- 노조비 = 200
- 동결비 = 300 (운영 값으로 확인 필요)

**확인 포인트 (formula 결과)**:
- [ ] 누적냉장료 = 0 또는 (오늘 - 최초입고일) × 1500 (이동입고일 fallback 작동)
- [ ] 판매원가 > 0 (단가 + 누적냉장료/총중량 + 비용/총중량)

## 시나리오 2 — 출고 신청 → 승인

**액션**: 위 LOT에서 3박스 출고 신청 → 관리자 승인

**확인 포인트 (출고 관리)**:
- [ ] 출고시점 단가
- [ ] 출고시점 냉장료
- [ ] 출고시점 입출고비
- [ ] 출고시점 노조비
- [ ] **출고시점 동결비** ← 핵심 (1단계 추가)
- [ ] 출고시점 판매원가 = 단가 + 냉장료 + 입출고비 + 노조비 + **동결비** + 이월 4개(=0)
- [ ] 출고시점 판매금액
- [ ] 출고시점 손익 = 판매금액 − 판매원가

**확인 포인트 (LOT별 재고)**:
- [ ] 재고수량 10 → 7로 차감

**예상 계산** (1박스 = 11kg, 보관 0일 가정):
- 단가 = 수매가 / 총중량 (kg당)
- 냉장료 = 0 (당일 출고)
- 입출고비/노조비/동결비 = LOT 저장값 그대로
- 판매원가 = 단가 + 0 + 500 + 200 + 300 = 단가 + 1000

## 시나리오 3 — 재고 이동 신청 → 승인

**액션**: 시나리오 1의 LOT 잔여 7박스 중 4박스를 부산냉동으로 이동 신청 → 관리자 승인

**확인 포인트 (새 LOT)**:
- [ ] LOT번호 새 채번 (이동일 prefix)
- [ ] **최초입고일 = 원본 LOT의 최초입고일** ← 핵심 (보존)
- [ ] **이동입고일 = 이동 승인일** ← 핵심 (이번에 신규 채움)
- [ ] 보관처 = 부산냉동
- [ ] 재고수량 = 4
- [ ] 수매가 = 원본 수매가 × (4/7) ≈ 0.57배 (반올림)
- [ ] **이월냉장료 = (원본 누적냉장료 + 원본 이월냉장료=0) × 4/7**
- [ ] **이월입출고비 = (500 + 0) × 4/7 ≈ 286**
- [ ] **이월노조비 = (200 + 0) × 4/7 ≈ 114**
- [ ] **이월동결비 = (300 + 0) × 4/7 ≈ 171**
- [ ] 새 보관처 비용: 냉장료단가/입출고비/노조비/동결비 = 부산냉동 값
- [ ] LOT번호(이동출처) = 원본 LOT 링크

**확인 포인트 (원본 LOT)**:
- [ ] 최초입고일 변경 X (보존)
- [ ] 재고수량 7 → 3으로 차감
- [ ] 이월 4개 = 0 (원본은 그대로)

**확인 포인트 (formula 검증, 새 LOT)**:
- [ ] 판매원가 = 단가 + (오늘 - 이동입고일 = 0일) × 냉장료단가/총중량 + (입출고비+노조비+동결비)/총중량 + (이월 4개)/총중량

## 시나리오 4 — 입고 반려

**액션**: 시나리오 1과 별개로 신규 입고 신청 → 승인 후 **반려**

**확인 포인트 (LOT별 재고)**:
- [ ] 재고수량 = 0 (soft delete)
- [ ] 냉장료단가 = null
- [ ] 입출고비 = null
- [ ] 노조비 = null
- [ ] **동결비 = null** ← 핵심 (1단계 추가)
- [ ] LOT 레코드 자체는 보존 (참조 유지)

---

## 시나리오 5 — 출고 반려

**액션**: 시나리오 2와 별개로 신규 출고 신청 → 승인 후 **반려**

**확인 포인트 (출고 관리)**:
- [ ] 출고시점 단가/냉장료/입출고비/노조비 = null
- [ ] **출고시점 동결비 = null** ← 핵심 (1단계 추가)
- [ ] 출고시점 판매원가/판매금액/손익 = null
- [ ] 승인상태 = "반려" + 반려사유 저장

**확인 포인트 (LOT별 재고 + 입고 관리)**:
- [ ] LOT.재고수량 + 출고수량으로 복구
- [ ] 입고관리.잔여수량 + 출고수량으로 복구

## 시나리오 6 — D1 재이동 (이미 이동된 LOT을 다시 이동)

**액션**: 시나리오 3에서 생성된 부산냉동의 새 LOT 4박스 중 2박스를 다시 한림냉동으로 이동 → 관리자 승인

**확인 포인트 (재이동 후 새 LOT)**:
- [ ] 최초입고일 = 시나리오 1의 LOT 최초입고일 (체인 보존)
- [ ] 이동입고일 = 재이동 승인일
- [ ] 재고수량 = 2
- [ ] 수매가 = 시나리오 3 LOT 수매가 × (2/4) = × 0.5
- [ ] **이월냉장료 = (시나리오 3 LOT의 누적냉장료 + 이월냉장료) × 0.5**
- [ ] **이월입출고비 = (600 + 286) × 0.5 = 443** (현재 보관처 부산냉동 비용 + 이월 합산)
- [ ] **이월노조비 = (250 + 114) × 0.5 = 182**
- [ ] **이월동결비 = (350 + 171) × 0.5 = 261**

이게 핵심: 두 번 이동된 LOT의 이월 경비는 모든 이전 보관처에서 발생한 비용이 합산되어 있어야 함.
