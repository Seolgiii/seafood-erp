# ERP 핵심 구조 (큰 그림)

> 수동 관리. /wrap-up은 파일이 없을 때만 초기화하며, 존재 시 절대 덮어쓰지 않음.
> 시스템 구조 변경 시 이 파일을 직접 편집.

## 시스템 전체 관계도

```mermaid
graph TD
    subgraph "비즈니스 핵심"
        입고[입고_관리]
        출고[출고_관리]
        이동[재고_이동]
        조회[재고_조회]
        지출[지출결의]
    end

    subgraph "허브"
        LOT[LOT별_재고]
        결재[결재_수신함]
        PDF[PDF_생성]
    end

    subgraph "인프라"
        PIN[PIN_인증]
        권한[서버_권한_검증]
        Idem[POST_Idempotency]
        AT[(Airtable)]
    end

    입고 --> LOT
    출고 --> LOT
    이동 --> LOT
    조회 --> LOT

    조회 --> 출고
    조회 --> 이동

    입고 --> 결재
    출고 --> 결재
    이동 --> 결재
    지출 --> 결재

    결재 --> PDF

    입고 -.-> 권한
    출고 -.-> 권한
    지출 -.-> 권한

    입고 -.-> Idem
    출고 -.-> Idem

    PIN -.-> 권한

    LOT -.-> AT
    결재 -.-> AT

    style LOT fill:#ffd700
    style 결재 fill:#ffa500
    style AT fill:#e0e7ff
```

화살표 의미:
- 실선: 직접 의존 / 데이터 흐름
- 점선: 권한 검증 / 부수 의존

## 핵심 시나리오 분포

```mermaid
graph LR
    subgraph "사용자 액션 시나리오 (A)"
        A1((A1 입고골든))
        A2((A2 출고골든))
        A3((A3 재고이동))
        A4((A4 지출결의))
        A5((A5 재고조회))
    end

    subgraph "데이터 흐름 (B)"
        B1((B1 LOT비용))
        B2((B2 출고손익))
        B3((B3 LOT일련번호))
    end

    A1 -.-> B1
    A2 -.-> B2
    A3 -.-> B1
```
