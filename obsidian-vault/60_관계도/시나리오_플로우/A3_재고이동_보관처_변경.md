> 마지막 갱신: 2026-05-08

## 시퀀스

```mermaid
sequenceDiagram
    participant 작업자
    participant 재고_조회
    participant 재고_이동
    participant POST_Idempotency
    participant 결재_수신함
    participant LOT별_재고
    participant 보관처_비용_이력

    작업자->>재고_조회: 1. 작업자가 재고 조회로 이동 대상 LOT 검색
    재고_조회->>재고_이동: 2. 다건 LOT 카트에 담기 (출고와 동일 카트 UX)
    재고_이동->>POST_Idempotency: 3. POST + Idempotency
    POST_Idempotency->>재고_이동: 4. 재고이동 레코드 생성 (펜딩)
    재고_이동->>결재_수신함: 5. 관리자 결재 수신함 검토
    결재_수신함->>LOT별_재고: 6. 승인 시 — 새 입고관리 레코드 생성 (비고="재고 이동") + 새 LOT 생성
    LOT별_재고->>보관처_비용_이력: 7. 새 보관처의 시점 비용 적용
    Note over 보관처_비용_이력: 8. 원본 LOT 차감 (재고수량 감소)
```
