# TRANSFER 반려 자동 복구

## 상태
✅ 해결 (2026-05-14)

## 한 줄 요약
TRANSFER 반려 시 안전 가드 3종 통과하면 자동 복구. 후속 작업(출고/재이동)이 있으면 차단 + [INTEGRITY-ALERT].

## 결정 내용
재고 이동(TRANSFER) 결재가 반려되면 원본 재고를 복구해 정합성을 유지한다.
다만 신규 LOT/입고관리에서 후속 작업이 일어났을 수 있어 위험도가 있으므로, 가드를
통과한 경우에만 자동 복구를 진행한다.

### 안전 가드 3종
1. **(a) 재고 변동 없음** — 신규 LOT.재고수량 == 이동수량 (출고·조정으로 변한 적 없음)
2. **(b) 재이동 없음** — 신규 LOT을 원본으로 한 재이동(반려 외) 부재
3. **(c) 출고 없음** — 신규 입고관리에서 발생한 출고(반려 외) 부재

### 통과 시 복구 동작
- 원본 LOT.재고수량 += 이동수량
- 원본 입고관리.잔여수량 += 이동수량
- 신규 LOT.재고수량 = 0 (soft delete — 레코드 보존)
- 신규 입고관리.잔여수량 = 0, 승인상태 = "반려"

### 차단 시
- [INTEGRITY-ALERT] 로그 + `success:false` → 반려 처리 자체 차단
- 운영자는 차단 사유(출고/재이동)를 먼저 정리한 후 다시 반려 시도

### Soft delete 선택 이유
입고 반려와 일관성. LOT/입고관리 hard delete는 link 깨질 위험. 화면에서는 재고수량 0이라
안 보이지만 Airtable에 이력 보존.

### 옵션 B 이월 4개
신규 LOT 재고수량이 0이 되면 손익 계산상 의미 없음 → 별도 클리어 불필요. 원본 LOT의
이월값은 이동 시 손대지 않았으므로 그대로 유지.
