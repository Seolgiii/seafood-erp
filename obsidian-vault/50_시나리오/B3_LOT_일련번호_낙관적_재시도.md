# B3 LOT 일련번호 낙관적 재시도

## 트리거
입고 승인 시점에 LOT 일련번호(YYMMDD-품목코드-규격-미수-전체일련번호) 발급

## 흐름
1. 입고 승인 → 영업일 9시 기준 일자 산출 → [[30_모듈별_상세/입고_관리]]
2. 같은 영업일의 마지막 일련번호 조회 (Airtable 최대값 +1) → [[30_모듈별_상세/LOT별_재고]]
3. LOT 레코드 생성 시도 (낙관적) → [[30_모듈별_상세/LOT별_재고]]
4. 동시 입고로 충돌 시 — 재시도 (다음 일련번호) → [[30_모듈별_상세/LOT별_재고]]
5. 검증~POST 사이 1ms race window 잔여 — 솔직히 문서화

## 모듈 간 데이터 흐름
- [[30_모듈별_상세/입고_관리]] → [[30_모듈별_상세/LOT별_재고]]: 영업일자 + 품목/규격/미수 → 일련번호 산출
- (race 시) [[30_모듈별_상세/LOT별_재고]] → [[30_모듈별_상세/운영_로거]]: 재시도 횟수 로그
