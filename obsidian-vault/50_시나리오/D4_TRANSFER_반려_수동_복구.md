# D4 TRANSFER 반려 수동 복구

## 트리거
재고 이동(TRANSFER) 결재가 반려될 때

## 흐름 (자동 복구 미구현 — 수동 보정)
1. 관리자가 TRANSFER 결재 반려 → [[30_모듈별_상세/결재_수신함]]
2. **자동 복구 미실행** — 새 입고관리 레코드 + 새 LOT은 그대로 잔존
3. 원본 LOT 재고는 차감된 상태 → 잔여 정합성 깨짐
4. [INTEGRITY-ALERT] prefix 로그 발생 → [[30_모듈별_상세/운영_로거]]
5. 운영자가 grep `[INTEGRITY-ALERT]`로 추적 → 수동으로 새 LOT 삭제 + 원본 LOT 복구 → [[30_모듈별_상세/재고_이동]]

## 모듈 간 데이터 흐름
- [[30_모듈별_상세/결재_수신함]] → [[30_모듈별_상세/재고_이동]]: 반려 상태
- [[30_모듈별_상세/재고_이동]] → [[30_모듈별_상세/운영_로거]]: [INTEGRITY-ALERT] prefix 로그
- (수동 보정) 운영자 → [[30_모듈별_상세/재고_이동]] / [[30_모듈별_상세/LOT별_재고]]: 잔여 정리
