# E2 일일 정산 이메일 (A4 인쇄)

## 트리거
Vercel Cron 매일 09:00 KST 발화 → 어제 처리분 정산 이메일

## 흐름
1. Cron 트리거 → `/api/cron/daily-report` 진입 → [[30_모듈별_상세/일일_정산_이메일]]
2. CRON_SECRET 검증 → [[30_모듈별_상세/서버_권한_검증]]
3. 어제 데이터 집계:
   - 입고 (입고일자=어제) → [[30_모듈별_상세/입고_관리]]
   - 출고 (출고일자=어제) → [[30_모듈별_상세/출고_관리]]
   - 재고이동 (이동일자=어제) → [[30_모듈별_상세/재고_이동]]
   - 지출 (지출일자=어제) → [[30_모듈별_상세/지출결의]]
4. 손익 추정 (출고 판매가 합 - 입고 수매가 합 - 지출 합) → [[30_모듈별_상세/일일_정산_이메일]]
5. 결재 대기 분리 (어제 신청 / 그 외 누적) + 24h 미처리 강조 → [[30_모듈별_상세/일일_정산_이메일]]
6. A4 인쇄 친화 CSS HTML 본문 작성
7. Resend API 호출 → 메일 발송 (CTA 링크 제거 — 도메인 mismatch 임시 fix)
8. ALERT_THRESHOLD 초과 시 강조 표시

## 모듈 간 데이터 흐름
- 4개 모듈 (입고/출고/이동/지출) → [[30_모듈별_상세/일일_정산_이메일]]: 어제 처리분 레코드
- [[30_모듈별_상세/일일_정산_이메일]] → Resend → SMTP: HTML 메일
