# D1 결재 양방향 토글 + 재고 자동 복구

## 트리거
관리자가 결재 수신함에서 승인 ↔ 반려 토글 (입고/출고/재고이동)

## 흐름
1. 관리자가 결재 카드 클릭 → 액션 시트(CompletedItemActionSheet) 노출 → [[30_모듈별_상세/결재_수신함]]
2. 양방향 변경 — 승인↔반려 또는 반려↔승인 → [[30_모듈별_상세/결재_수신함]]
3. 결재 타입별 자동 복구:
   - 입고 반려 — LOT 재고수량=0 + 보관처 비용 3필드 null (soft delete) → [[30_모듈별_상세/입고_관리]] / [[30_모듈별_상세/LOT별_재고]]
   - 입고 재승인 — `createLotOnInboundApproval` 재실행, LOT 복원 → [[30_모듈별_상세/입고_관리]] / [[30_모듈별_상세/LOT별_재고]]
   - 출고 반려 — 잔여수량/LOT재고 +outQty 복구 + 출고시점 비용 7필드 null → [[30_모듈별_상세/출고_관리]] / [[30_모듈별_상세/LOT별_재고]]
4. TRANSFER 양방향은 미구현 — [INTEGRITY-ALERT] 로그 + 수동 보정 → [[30_모듈별_상세/재고_이동]] / [[30_모듈별_상세/운영_로거]]

## 모듈 간 데이터 흐름
- [[30_모듈별_상세/결재_수신함]] → [[30_모듈별_상세/입고_관리]] / [[30_모듈별_상세/출고_관리]] / [[30_모듈별_상세/재고_이동]]: 새 상태 (승인/반려)
- 각 모듈 → [[30_모듈별_상세/LOT별_재고]]: 재고 차감/복구 또는 soft delete
