# A1 입고 골든패스

## 트리거
작업자가 입고 폼 제출 → 관리자가 결재 수신함에서 승인

## 흐름
1. 작업자가 입고 폼 입력 (품목/규격/미수/보관처/매입처) → [[30_모듈별_상세/입고_관리]]
2. 입력 sanitize + zod 검증 → [[30_모듈별_상세/입력_Sanitize_Schema]]
3. POST + X-Idempotency-Key 헤더 → [[30_모듈별_상세/POST_Idempotency]]
4. 입고 레코드 생성 (상태=펜딩, LOT 미생성) → [[30_모듈별_상세/입고_관리]]
5. 작업자가 신청 내역에서 확인 → [[30_모듈별_상세/신청_내역]]
6. 관리자가 결재 수신함에서 검토 → [[30_모듈별_상세/결재_수신함]]
7. 승인 시 — LOT 자동 생성 (일련번호 발급, 영업일 9시 기준) → [[30_모듈별_상세/LOT별_재고]]
8. 보관처 비용 이력에서 시점 단가 lookup → LOT 비용 3필드 set → [[30_모듈별_상세/보관처_비용_이력]]
9. 입고증 PDF 생성 + LOT QR 임베드 → [[30_모듈별_상세/PDF_생성]]
10. PDF Vercel Blob 업로드, 입고관리.PDF필드 set

## 모듈 간 데이터 흐름
- [[30_모듈별_상세/입고_관리]] → [[30_모듈별_상세/LOT별_재고]]: 신규 LOT (일련번호, 품목, 보관처, 수량)
- [[30_모듈별_상세/보관처_비용_이력]] → [[30_모듈별_상세/LOT별_재고]]: 시점 단가 3필드 (냉장료/입출고비/노조비)
- [[30_모듈별_상세/입고_관리]] → [[30_모듈별_상세/PDF_생성]]: 입고증 데이터 + LOT 번호 → PDF Buffer
- [[30_모듈별_상세/PDF_생성]] → Vercel Blob: PDF 바이너리
