# E1 LOT QR 평생 식별자

## 트리거
입고 승인 시점 — LOT 생성과 함께 QR이 입고증 PDF에 임베드 (평생 식별자)

## 흐름
1. 입고 승인 → LOT 자동 생성 (일련번호 발급) → [[30_모듈별_상세/입고_관리]] / [[30_모듈별_상세/LOT별_재고]]
2. 입고증 PDF 생성 — `generateInboundPdf`에서 LOT번호 → QR PNG data URL 인코딩 → [[30_모듈별_상세/PDF_생성]]
3. QR을 입고증 PDF 하단에 삽입 → PDF Buffer 반환
4. 박스에 입고증 부착 (QR 노출, 평생 식별자)
5. (계획) 출고증 PDF에도 같은 QR 삽입 — 같은 LOT이니까
6. 작업자가 QR 스캔 → 통합 LOT 페이지로 라우팅 (계획) → [[30_모듈별_상세/QR_스캔]]
7. (현재) `/inventory/outbound?lot={LOT번호}` 출고 페이지로 직접 라우팅 → [[30_모듈별_상세/출고_관리]]
8. (계획) 통합 LOT 페이지(/lot/{번호}) — 권한·상태별 액션 분기 (조회 / 출고 / 이동) → [[30_모듈별_상세/재고_조회]]

## 구현 상태
- **현재**: 입고증 PDF에만 QR 임베드. 출고증 PDF엔 QR 없음 (코드 검증 결과)
- **현재 라우팅**: 입고증 QR → `/inventory/outbound?lot=...` (출고 페이지)
- **계획**:
  - 출고증 PDF에도 같은 QR 추가
  - 통합 LOT 랜딩 페이지(/lot/{번호}) 신규 개발 → 권한·상태 분기
  - 마이그레이션: 새로 발행되는 PDF부터 적용
