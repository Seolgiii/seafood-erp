# A4 지출 결의 골든패스

## 트리거
작업자(또는 관리자)가 지출 결의 신청 → 권한 분기 → 승인 → 지출결의서 PDF

## 흐름
1. 작성자가 지출 결의 폼 입력 (금액/카테고리/소속/직급) → [[30_모듈별_상세/지출결의]]
2. 영수증 업로드 (Vercel Blob) → [[30_모듈별_상세/지출결의]]
3. 입력 sanitize + zod 검증 → [[30_모듈별_상세/입력_Sanitize_Schema]]
4. POST + Idempotency → [[30_모듈별_상세/POST_Idempotency]]
5. 서버 액션 진입 시 작업자 role 재검증 → [[30_모듈별_상세/서버_권한_검증]]
6. 100만원 분기 — amount > 1,000,000 시 role === MASTER 검증 → [[30_모듈별_상세/서버_권한_검증]]
7. 작성자 신청 내역에 표시 → [[30_모듈별_상세/신청_내역]]
8. 관리자 결재 수신함에서 검토 → [[30_모듈별_상세/결재_수신함]]
9. 승인 시 지출결의서 PDF 생성 (인감 서명 포함) → [[30_모듈별_상세/PDF_생성]]
10. PDF Vercel Blob 업로드

## 모듈 간 데이터 흐름
- [[30_모듈별_상세/지출결의]] → [[30_모듈별_상세/서버_권한_검증]]: amount + 작업자 ID → role 매칭
- [[30_모듈별_상세/지출결의]] → [[30_모듈별_상세/PDF_생성]]: 결의서 데이터 → PDF Buffer
