# C2 PostgreSQL 이전 파급 영향

## 트리거
Airtable 한계(레코드 수 / 응답 속도 / 동시 사용자) 도달 시 PostgreSQL 이전 검토

## 흐름 (의사결정 의존 — 마이그레이션 영향 범위)
1. 트리거 조건 만족 → 이전 결정 → [[40_결정기록/PostgreSQL_이전_시점]]
2. 데이터 모델링 — 7개 운영 테이블 + 4개 마스터 테이블 PostgreSQL 스키마 변환
3. 어댑터 모듈 교체 (lib/airtable.ts → 새 어댑터)
4. 영향 모듈 다수 — 입고/출고/재고이동/LOT/지출/보관처 비용 모두 데이터 어댑터 의존
5. AI 데이터 분석 활용 가능성 (분석 도구 연결 용이)
